# 🤝 Contributing to Luca OS

Welcome to the L.U.C.A workforce. We welcome contributions from engineers, hackers, and researchers.

## 🗺️ Roadmap

Check our [Active Tasks](https://github.com/your-org/luca-os-prime/issues) for high-priority features and bug fixes.

## 🛠️ Development Workflow

1. **Fork & Branch**: Create a feature branch from `main`.
   ```bash
   git checkout -b feature/your-feature-name
   ```
2. **Code Standards**: We use ESLint and Prettier. Run `npm run lint` before committing.
3. **Commit Messages**: Use conventional commits (e.g., `feat: added network intercepts`, `fix: resolved adb timeout`).
4. **Testing**: Add units tests in `src/__tests__` and run `npm run test`.

## 🛡️ Pull Request Process

- Ensure `npm run safety-check` passes locally.
- Provide a clear description of the change and any tactical impact.
- Reference any related issues.
- All PRs require review from a Prime Instance (Core Maintainer).
